# ATIVIDADE – ANÁLISE DE QUALIDADE DE CÓDIGO COM ESLINT

## Disciplina
Qualidade de Software

## Objetivo
Compreender o papel da análise estática de código no processo de garantia da qualidade de software, utilizando a ferramenta ESLint para identificar, analisar e corrigir problemas em aplicações desenvolvidas em JavaScript ou TypeScript.

## Contexto
Durante o desenvolvimento de software, diversos problemas podem surgir antes mesmo da execução do sistema, como:

- erros de programação;
- inconsistências de estilo;
- más práticas de desenvolvimento;
- código de difícil manutenção;
- vulnerabilidades causadas por descuidos de implementação.

Ferramentas de análise estática ajudam a detectar esses problemas automaticamente, aumentando a qualidade do produto e reduzindo custos de manutenção.

Uma das ferramentas mais utilizadas para esse fim no ecossistema JavaScript e TypeScript é o ESLint.

Nesta atividade você deverá utilizar o ESLint em um projeto próprio, analisar os resultados obtidos e produzir evidências do processo de melhoria do código.

## Parte 1 – Desenvolvimento da Aplicação
Desenvolva uma aplicação própria utilizando:

- Node.js + JavaScript  
ou  
- Node.js + TypeScript

A aplicação deve possuir complexidade suficiente para permitir uma análise significativa pelo ESLint.

**Requisitos mínimos:**
- mínimo de 5 arquivos fonte;
- mínimo de 300 linhas de código;
- utilização de funções;
- utilização de estruturas condicionais;
- utilização de estruturas de repetição;
- utilização de módulos/importações;
- interação com entrada de dados ou arquivos.

O tema da aplicação é livre.

**Exemplos:**
- sistema de biblioteca;
- controle financeiro;
- agenda de contatos;
- gerenciamento de estoque;
- sistema de pedidos;
- cadastro de alunos;
- sistema de reservas.
